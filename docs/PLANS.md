## 플랜 개요

### 현재 구상

| 플랜 | 가격 | 메시지 이력 | 핵심 차별점 |
|------|------|------------|------------|
| **FREE** | $0/월 | 7일 보관 | 기본 기능, 1개 세션 |
| **PRO** | $8.99/월 | 무제한 (아카이빙) | 무제한 세션, 멀티 디바이스 |
| **TEAM** | 미정 | 미정 | 팀 협업, 권한 관리 |

### 설계 원칙

1. **FREE는 충분히 유용해야 함**: 7일이면 대부분의 단기 프로젝트 커버
2. **PRO 전환 동기 명확**: 장기 프로젝트, 이력 보존 필요 시 자연스러운 업그레이드
3. **스토리지 비용 관리**: 무제한이라도 아카이빙으로 비용 최적화

## 기능 매트릭스

### FREE 플랜 ($0/월)

| 기능 | 제한 | 이유 |
|------|------|------|
| PC 세션 | 1개 | 개인 사용자 기본 |
| 디바이스 페어링 | 1대 | 단일 기기 사용 |
| 일일 메시지 | 100개 | API 남용 방지 |
| **메시지 이력** | **7일 보관** | 스토리지 비용 관리 |
| E2E 암호화 | 포함 | 핵심 기능 |
| 지원 | 커뮤니티 | 비용 절감 |

**7일 선택 이유**:
- 짧은 코딩 세션(버그 수정, 간단한 기능)은 7일 내 완료
- 충분히 유용하면서도 장기 사용자에게 PRO 전환 동기 부여
- 스토리지: 유저당 ~350MB/월 → 7일이면 ~80MB로 제한

### PRO 플랜 ($8.99/월)

| 기능 | 제한 | 이유 |
|------|------|------|
| PC 세션 | 무제한 | 파워 유저 |
| 디바이스 페어링 | 3대 | 폰 + 태블릿 + 다른 PC |
| 일일 메시지 | 무제한 | 제한 없음 |
| **메시지 이력** | **무제한 (아카이빙)** | 장기 프로젝트 지원 |
| 푸시 알림 | 포함 | 편의 기능 |
| 우선 지원 | 포함 | 가치 차별화 |

**무제한 + 아카이빙 전략**:
- 활성 세션: 최근 30일 메시지는 빠른 조회
- 아카이브: 30일 이상 지난 세션은 압축 저장
- 복원: 아카이브된 세션 접근 시 온디맨드 복원

## 스토리지 최적화 전략

### 1. FREE 플랜: 7일 만료 삭제

```
┌─────────────────────────────────────────────────────┐
│  FREE 플랜 메시지 라이프사이클                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Day 0     Day 7      삭제                          │
│  ──────────────────────→                            │
│  [메시지 생성] [보관 중] [cron job으로 삭제]          │
│                                                     │
│  스토리지: ~80MB/유저 (7일 * ~500KB/일)              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**구현 방법**:
```sql
-- 매일 실행: 7일 지난 FREE 유저 메시지 삭제
DELETE FROM messages
WHERE session_id IN (
  SELECT s.id FROM sessions s
  JOIN users u ON s.user_id = u.id
  JOIN subscriptions sub ON u.id = sub.user_id
  WHERE sub.plan = 'free'
)
AND created_at < NOW() - INTERVAL '7 days';
```

### 2. PRO 플랜: 30일 이후 아카이빙

```
┌─────────────────────────────────────────────────────────────────┐
│  PRO 플랜 메시지 라이프사이클                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Day 0          Day 30           영구 보관                      │
│  ──────────────────────────────────────→                        │
│  [활성 저장]    [아카이브로 이동]  [압축 스토리지]                │
│  (빠른 조회)    (읽기 전용)        (온디맨드 복원)               │
│                                                                 │
│  활성 스토리지: ~15MB/유저 (30일)                                │
│  아카이브 스토리지: 별도 관리 (저비용)                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**아카이빙 방식 옵션**:

| 방식 | 장점 | 단점 | 비용 |
|------|------|------|------|
| **A. DB 내 압축** | 단순 | 압축률 낮음 | 중간 |
| **B. S3/R2 이동** | 저렴, 확장성 | 구현 복잡 | 낮음 |
| **C. 세션 단위 JSONL** | 복원 빠름 | 관리 복잡 | 낮음 |

**추천: B. S3/R2 이동**
- Cloudflare R2: 무료 egress, $0.015/GB/월
- 30일 지난 세션 → JSONL로 압축 → R2 업로드 → DB에서 삭제
- 조회 시: R2에서 다운로드 → 캐시 → 표시

### 3. 비용 예측 (아카이빙 적용 시)

| 규모 | 활성 DB | 아카이브 (R2) | 총 비용 |
|------|---------|--------------|--------|
| 1,000 유저 | ~1GB | ~5GB | $0.08/월 |
| 5,000 유저 | ~5GB | ~25GB | $0.38/월 |
| 10,000 유저 | ~10GB | ~50GB | $0.75/월 |
