# Pocket AI 설계 개선 제안서 (Design Improvement Proposal)

본 문서는 Pocket AI 프로젝트의 핵심 목표인 **"Simple(단순함)"**과 **"Convenience(편리함)"**을 극대화하기 위해 기존 설계 문서를 리뷰하고 도출한 개선안입니다.

---

## 1. 아키텍처 및 인프라 단순화 (Simplicity)

### 1.1 PostgreSQL(DB) 구조의 경량화 (구독 모델 대비)
- **현재 설계**: 순수 릴레이 서버임에도 불구하고 사용자, 세션, 기기 관리를 위해 PostgreSQL과 Prisma를 사용.
- **문제점**: 초기 MVP 단계에서 DB 설정 및 Prisma 마이그레이션이 다소 무겁게 느껴질 수 있음.
- **사용자 피드백 반영 (구독제 대비)**:
  - 향후 구독제(Pro/Team 등) 도입을 고려할 때, 사용자 결제 상태, 디바이스 등록 권한, 팀 단위 권한 관리를 위해서는 **관계형 DB(PostgreSQL) 유지가 필수적**임.
  - **개선 제안 (단순화)**: DB는 유지하되, 무거운 ORM(Prisma) 대신 가벼운 Query Builder(예: Kysely)를 사용하거나 스키마를 최소화하여 운영 부담을 줄임. 
  - 세션 연결 상태 메시지나 채팅 히스토리는 절대 DB에 저장하지 않고(메모리/Redis 처리), DB는 오직 **사용자, 결제 정보, 기기 등록 상태**만 관리하는 구조를 엄격히 지킴.

### 1.2 패키지 구조 통합
- **현재 설계**: `@pocket-ai/cli`(호스트용)와 `@pocket-ai/agent`(원격 제어용) 패키지가 분리되어 있음.
- **문제점**: 사용자가 용도에 따라 2개의 패키지 존재를 알아야 하고 각각 관리해야 함.
- **개선 제안**:
  - 단일 `@pocket-ai/cli` 패키지로 통합.
  - `pocket-ai start` (호스트 띄우기)
  - `pocket-ai remote` (터미널에서 원격 접속하기)
  - 사용자는 하나의 패키지만 알면 됨.

---

## 2. 사용자 경험 및 편리함 극대화 (Convenience)

### 2.1 QR 코드 스캔 제거 및 '계정 기반 자동 연결'
- **현재 설계**: 모바일 PWA에서 PC 화면의 QR 코드를 스캔하여 기기 페어링 및 암호화 키를 교환함.
- **문제점**:
  - 매번 의자에서 일어나 폰의 카메라로 모니터를 찍어야 하는 번거로움 발생.
  - 외부나 카페 등 카메라 활용이 불편한 상황, 혹은 노트북에서 다른 노트북으로 원격 제어할 때 큰 불편.
- **개선 제안 (비대칭키 기반 E2E 암호화 + 로그인 자동 탐색)**:
  1. 사용자가 PC에서 `pocket-ai login` 입력 (GitHub OAuth 브라우저 로그인).
  2. 사용자가 폰/PWA에서도 같은 계정으로 로그인.
  3. 로그인 즉시 PWA 화면에 **"활성화된 내 PC 목록"**이 자동으로 노출됨 (원클릭 연결).
  4. **보안 (QR 대체)**: PC가 시작될 때 임시 비대칭키(Keypair)를 생성하여 공개키(Public Key)를 서버에 등록해둠. 모바일 PWA에서 접속 시, 대칭키(AES)를 랜덤 생성한 뒤 PC의 공개키로 암호화하여 서버를 거쳐 PC로 전달함. 이를 통해 QR 스캔 없이도 서버가 풀 수 없는 완벽한 E2E 암호화가 자동 체결됨.

### 2.2 데몬(Daemon) 제어의 가시성 확보
- **현재 설계**: 터미널이 닫혀도 세션을 유지하는 데몬 프로세스 존재.
- **문제점**: 터미널이 닫힌 상태에서 백그라운드 프로세스를 어떻게 끄거나 상태를 볼지 직관적이지 않음 (서버에 계속 연결되어 배터리/자원 소모 가능성).
- **개선 제안**:
  - `pocket-ai status` 명령어로 현재 돌고 있는 데몬 상태 확인.
  - `pocket-ai stop` 명령어로 명시적인 데몬 종료 지원.
  - **PWA 원격 제어 기능**: 폰(PWA) 화면상에서도 집에 켜진 PC의 데몬 세션을 원격으로 로그아웃(종료) 시킬 수 있는 버튼 제공.

### 2.3 Web(PWA) UI/UX 고도화 및 직관성 확보
- **문제점**: 현재 대시보드와 터미널 화면이 MVP 수준으로 구현되어 있으며, 모바일(PWA) 환경이나 장시간 사용 시 세밀한 조작성이나 연결 상태 피드백이 부족합니다.
- **개선 제안 (UI_ARCHITECTURE.md 반영)**:
  - **로그인 & 온보딩 개선**: OAuth 에러 핸들링(실패/거부) 구체화 및 비로그인 사용자 가이드 추가로 초기 진입 장벽 완화 (WhatsApp Web 스타일의 단순 진입 레이아웃 적용 포함).
  - **실시간 반응형 대시보드 (Zero-Delay)**: 10초 주기 폴링을 제거하고 Socket.IO 기반 실시간 세션 상태(Online/Offline/Ping 등 마지막 활동 시간) 동기화 도입. 이와 함께 특정 호스트명이나 AI 엔진 필터 검색 기능을 추가.
  - **세션 제어 확장 (모달/사이드패널)**: CLI 터미널에 굳이 들어가지 않더라도, 대시보드 목록에서 세션 복제/공유/종료 및 요약 로그 보기가 가능한 퀵 액션 모달 지원.
  - **터미널 사용성 극대화 (Mobile-Friendly)**: 좁은 화면을 위한 전체화면 모드 토글, 클립보드 완벽 지원(복사/붙여넣기), 스크롤 자동 위치 고정, 자동 연결 끊김 감지 및 재연결, 터미널 전체 로그 보존/다운로드.
  - **직관적 설정 화면 신설 (`/settings`)**: 가장 필수적인 역할인 "CLI 로그인 토큰 관리" (복사, 수명 확인, 재발급) 인터페이스를 신설하고, 기타 PWA 디스플레이 테마 등 사용자 맞춤 설정 환경 제공.

## 3. 데몬 및 세션 수명주기 단순화 (API & Security)

### 3.1 머신(Machine) API의 간소화와 다중 세션 지원
- **사용자 피드백 반영**: PC 한 대에서도 여러 프로젝트나 여러 AI(Claude, Gemini 등)를 위해 여러 개의 CLI 터미널(세션)을 동시에 띄워야 할 필요성이 큽니다.
- **기존 문제점**: 데몬이 머신을 등록하고(`POST /api/machines`), 그 머신 아래에 세션들을 배정하는 `/api/machines`, `/api/sessions`의 이중 API 구조가 지나치게 무겁습니다.
- **개선 제안**:
  - 다중 터미널 창(세션)은 완벽히 지원하되, 서버 API에서는 무거운 **`Machine` 개념 자체를 삭제**합니다.
  - PC에서 `pocket-ai start`를 할 때마다 독립적인 1개의 세션으로 간주하고, `/api/sessions` API 하나로만 등록합니다. 즉, 각 CLI 창이 독립된 주체로 동작하며 서버 입장에서는 수많은 'Session'들만 관리하면 되므로 백엔드 구조가 절반 이하로 단순해집니다.

### 3.2 세션 키 복구 흐름 제거
- **현재 설계**: 데몬 프로세스 재시작 시 서버에 암호화 보관된 키를 HSM 인증 등으로 복구하려 함 (`/api/sessions/:sessionId/recover-key`).
- **문제점**: 너무 무거운 엔터프라이즈급 인프라가 필요함.
- **개선 제안**:
  - 데몬이 죽으면, 억지로 키를 백업/복구하려 하지 말고 "해당 세션을 안전하게 종료/리셋"합니다.
  - 다음 로그인/기기 접속 시 새로운 대칭키를 즉시 생성·교환하여 이어가게끔 구조를 초경량 무상태(Stateless)로 만듭니다.

## 4. [추가 제안] 유료 구독(Pro/Team)을 이끌어낼 "Killer Features"

타사 원격 제어 프로그램(TeamViewer, AnyDesk)이나 단순 SSH 클라이언트와 확실하게 차별화되고, 월 $8.99를 기꺼이 지불하게 만들 "킥(Kick)"이 될 만한 프리미엄 기능들을 제안합니다.

### 4.1 "Session Replay" (타임머신 기능) - Pro Tier
- **개념**: 폰으로 작업하다가 실수를 했거나, 이전 코드가 기억 안 날 때 **CLI 화면을 스크롤/재생해서 과거 상태로 되돌리는 기능**입니다.
- **가치**: 좁은 모바일 화면에서는 `git diff`나 터미널 스크롤본을 읽기가 아주 고통스럽습니다. 이를 PWA 상에서 직관적인 UI(타임라인 바)로 제공하여 "아까 낮에 데스크톱에서Claude가 짜준 코드"로 쉽게 돌아갈 수 있게 해줍니다. 
- **구현 아이디어**: 서버에 평문 로그를 남기지 않고, PWA 클라이언트 단의 IndexedDB에 암호화된 상태로 최근 N시간의 터미널 상태를 저장해두고 스와이프만으로 타임라인을 넘나들게 합니다.

### 4.2 "Universal Clipboard & File Drop" (클립보드/파일 연속성) - Pro Tier
- **개념**: 폰(PWA)에서 복사한 텍스트나 사진이 PC(데몬)의 클립보드로 즉시 동기화되거나, 반대로 PC의 에러 로그가 폰 클립보드에 바로 들어오는 기능입니다.
- **가치**: 폰에서 구글링해서 찾은 코드를 폰의 터미널 앱에 욱여넣는 건 너무 힘듭니다. 하지만 이 기능이 있으면 폰 웹브라우저에서 텍스트를 복사한 뒤, Pocket AI PWA의 터미널에 붙여넣기만 하면 PC에 완벽하게 입력됩니다. (Apple의 Handoff/유니버설 클립보드와 유사한 경험)
- **차별점**: 기존 SSH 앱들은 모바일 환경의 클립보드 처리가 매우 조잡합니다. 이 끊김없는 연속성은 엄청난 편의를 줍니다.

### 4.3 "AI Agent Supervisor" (백그라운드 감시 및 알림) - Pro Tier
- **개념**: 단순히 터미널을 중계하는 것을 넘어, CLI에서 도는 `Claude Code`나 `Gemini` 에이전트의 구동 상태를 서버가 모니터링하다가 **에이전트가 에러를 뿜거나 사용자 입력(승인)을 기다릴 때만 폰으로 푸시 알림**을 보내줍니다.
- **가치**: "AI한테 리팩토링 시켜놓고 난 침대에 누워있어야지"가 완벽하게 성립됩니다. 계속 폰 화면을 켜고 터미널이 올라가는 걸 볼 필요 없이, AI가 질문할 때만 알림이 오므로 진정한 "원격 자동화"의 가치를 제공합니다.

### 4.4 "Shared Context" (팀원 간 AI 터미널 공유) - Team Tier
- **개념**: 특정 세션(URL 링크)을 팀원에게 읽기 전용(또는 쓰기 권한 부여)으로 공유합니다.
- **가치**: 주니어 개발자가 "AI가 이상한 코드를 짰는데 어디서부터 꼬인 건지 모르겠어요" 할 때, 시니어 개발자가 해당 링크로 들어와서 AI가 대답한 히스토리와 현재 CLI 터미널 창을 실시간으로 같이 봅니다. TeamViewer보다 훨씬 가볍고 개발 친화적인 페어 프로그래밍/디버깅 도구가 됩니다.

## 5. 요약 및 Action Item

이 프로젝트의 핵심은 **"설정에 신경 쓸 필요 없이, 로그인 한 번에 폰과 PC가 연결되어 코딩의 맥락이 이어지는 것"**입니다.

1. **[필수] 아키텍처 개선**: 가벼운 Query Builder(Kysely)를 도입하고, PostgreSQL은 메시지가 아닌 사용자/결제 정보 전용으로만 사용합니다.
2. **[필수] 통신 흐름 방식 변경**: QR 방식의 페어링은 보조 수단으로 내리고, 계정 기반 자동 매칭 + 비대칭키 암호화(E2E)를 중심으로 설계합니다.
3. **[필수] API 단순화 (다중 세션 지원)**: 다중 터미널 창(세션)은 완벽히 지원하되, 복잡한 `Machine` 등록 API는 삭제하고 모두 독립된 `Session`으로 단순하게 관리합니다. 서버의 세션 키 보관/복구 로직은 전면 제거합니다.
4. **[선택] 패키지 통합**: `@pocket-ai/cli` 단일 패키지로 합치고, 명령어를 직관적으로 통합(start/remote/status/stop)합니다.

이 제안들을 반영하면 아키텍처가 훨씬 덜 무겁고(Simple), 유저는 QR 코드를 찍는 수고 없이 원클릭 연결(Convenience)이 가능해집니다. 여기에 **Pro/Team 전용 킬러 피처("Session Replay", "AI Supervisor 알림")**를 더하면 수익성 높은 비즈니스 모델도 확립됩니다.
