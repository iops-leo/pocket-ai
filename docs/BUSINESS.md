## 비즈니스 개요

### 핵심 가치 제안

**"어디서든 PC의 AI 코딩 세션을 이어서 사용하세요"**

개발자가 출퇴근길, 회의 중, 외출 시에도 PC에서 실행 중인 Claude Code 세션을 모바일로 이어서 사용할 수 있습니다.

### 타겟 고객

| 세그먼트 | 특성 | 니즈 |
|---------|-----|-----|
| **개인 개발자** | 프리랜서, 사이드 프로젝트 | 이동 중 코딩 연속성 |
| **스타트업 개발자** | 바쁜 일정, 멀티태스킹 | 효율성, 빠른 응답 확인 |
| **기업 개발팀** | 협업, 보안 중요 | 팀 공유, 감사 로그 |
| **DevOps 엔지니어** | 서버 모니터링, 긴급 대응 | 어디서든 CLI 접근 |

### 시장 기회

- AI 코딩 도구 시장 급성장 (2025년 예상 $20B+)
- Claude Code, GitHub Copilot CLI 등 CLI 기반 도구 증가
- 원격/하이브리드 근무 지속
- 모바일 개발 생산성 도구 수요

## 가격 정책

### Freemium 모델

> **설계 원칙**: 서버는 E2E 암호화된 메시지를 저장하지만 복호화 키를 보유하지 않습니다 (Zero-Knowledge). 메시지는 암호화된 상태로 DB에 저장되어 세션 재접속 시 대화 이력을 복원할 수 있습니다. 차별점은 세션 수, 디바이스 페어링, 메시지 보관 기간, 팀 협업 기능으로 구분합니다.

```

│                         가격 티어                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  FREE                  PRO                      TEAM                │
│  $0/월                 $8.99/월                 $39/팀/월           │
│                        (연간 $7.49/월)          (5명 기준)          │
│                                                                     │
│  ✓ 1개 PC 세션         ✓ 무제한 PC 세션         ✓ PRO 모든 기능    │
│  ✓ 디바이스 1대 페어링  ✓ 디바이스 3대 페어링    ✓ 디바이스 10대    │
│  ✓ 100 메시지/일       ✓ 무제한 메시지          ✓ 팀 공유 세션     │
```

**푸시 알림 (PRO/TEAM)**: 세션 완료, 에러 발생 시 모바일로 즉시 알림

**팀 공유 세션 (TEAM)**: 하나의 PC 세션을 팀원들이 함께 관찰하거나 입력 가능

**권한 관리 (TEAM)**: 팀원별 읽기전용 / 입력가능 권한 설정

## 인프라 비용 분석

### 단계별 인프라 비용

```
┌───────────────────────────────────────────────────────────────────────┐
│                       월간 인프라 비용 예측                            │

│  초기성장       100-1K      Railway Hobby ($5)             $5-10       │
│  성장          1K-5K       Railway Pro + Supabase Pro     $45-55      │
│  확장          5K-10K      Railway Pro + Supabase Pro     $55-80      │
│  대규모        10K+        Railway Team + Supabase Pro    $100-200    │

└───────────────────────────────────────────────────────────────────────┘
```

> **비용 구조 변경 (v2.0 - 메시지 저장 도입)**:
> 1. **메시지 저장**: E2E 암호화된 메시지를 DB에 저장 → 대화 이력 복원 가능
> 2. **DB 사용량 증가**: 메시지 1개 ~500B, 100 메시지/일/유저 → ~1.5MB/월/유저
> 3. **보관 기간 제한**: 티어별 7/30/90일 → 스토리지 비용 관리
> 4. **Zero-Knowledge**: 서버는 암호화된 ciphertext만 저장, 복호화 불가능

### 손익분기점 분석

```
월 고정비용: $1 (Railway 무료 + Supabase 무료 + 도메인)
PRO 가격: $8.99/월
Stripe 수수료: $0.30/건
서버 한계비용: $0.025/유저 (메시지 저장 포함)

PRO 기여이익: $8.99 - $0.30 - $0.025 = $8.665/월

손익분기: 1 / 8.665 ≈ 1명의 PRO 유저

5% 전환율 가정 시:
필요한 총 사용자: 1 / 0.05 = 20명
```

> **메시지 저장 영향**: 한계비용이 $0.005 → $0.025로 증가했지만, PRO 가격($8.99) 대비 미미한 수준. 대화 이력 복원 기능이 유료 전환 동기 부여 → 전환율 상승으로 상쇄 예상. 손익분기점은 여전히 **20명** 수준.

## 결론

Pocket AI는 다음과 같은 비즈니스 모델로 지속 가능한 성장을 추구합니다:

1. **Freemium**: 무료로 사용자 확보, PRO/Team으로 수익화
2. **비용 최적화**: 사용자당 $0.04 이하 유지 (보관 기간 제한으로 스토리지 관리)
3. **점진적 확장**: 수요에 맞춰 인프라 확장
4. **커뮤니티 중심**: 바이럴 마케팅으로 CAC 최소화
5. **Enterprise 진입**: 고가치 고객으로 ARPU 상승
6. **Zero-Knowledge**: E2E 암호화 + 서버 복호화 불가 → 프라이버시 강점

핵심은 **단순함 유지**, **비용 효율성**, **대화 연속성**입니다.
