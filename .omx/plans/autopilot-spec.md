# Autopilot Spec — New Session 생성 버튼 무반응 버그

## 문제 요약
- PWA 대시보드의 `새 세션` 모달에서 `세션 생성` 클릭 시 실제 생성 로직이 실행되지 않음.
- 현재 구현은 `console.log` 후 모달만 닫는 placeholder 상태.

## 사용자 기대 동작
1. 사용자가 작업 디렉토리(cwd)와 엔진을 선택 후 생성 버튼 클릭
2. 서버에 세션 생성 요청이 전달됨
3. 요청 실패 시 사용자에게 명확한 에러 표시
4. 요청 성공 시 세션 목록이 갱신됨

## 범위
- PWA 대시보드 `handleNewSession` 실제 API 연동
- 서버 세션 목록 조회에서 사용자 세션(online/offline) 모두 반환
- 생성 직후 목록에서 세션을 확인 가능하도록 정렬 개선(online 우선)

## 비범위
- 원격 PC에서 실제 프로세스를 자동 실행하는 데몬 오케스트레이션
- QR 페어링/디바이스 링크 플로우 구현

## 수용 기준
- `세션 생성` 클릭 시 `/api/sessions` POST 호출됨
- 실패 시 모달 내 에러 메시지 표시
- 성공 시 세션 목록이 즉시 갱신됨
- GET `/api/sessions`가 online/offline 세션 모두 반환함
